Resumen del problema y solución solicitada


El Problema

Se desea proveer a una asociación que se dedica a la actividad de excursiones o visitas a sitios turísticos. Se requiere la administración de un flotilla de microbuses para las excursiones, diversas opciones de viaje con una cantidad máxima de personas que al mismo tiempo definirá la cantidad de microbuses para dicha actividad. Además se desea administrar información relacionada con las excursiones como: precio, servicios, reservaciones, condiciones de pago y cancelación. Al mismo tiempo se requiere que el cliente pueda consultar sobre excursiones que se ofrecen, realizar reservaciones, o cancelar reservas.

Solución solicitada

Desarrollar una aplicación web que proporcione una solución desde un entorno de administración que permita:

  • Gestionar el mantenimiento de la flotilla.
  • Definir y administrar los sitios turísticos.
  • Definir y administrar datos de los asociados y sus microbuses
  • Generar reportes para obtener los datos de contactos de clientes interesados en una excursión específica
Además se debe implementar una página de interacción con el cliente en el que se pueda:
  • Consultar excursiones, sitios, imágenes e información relacionada.
  • Realizar reservaciones en las excursiones por medio de formularios
  • Consultar reservaciones y cancelar, o cancelar la reserva

Justificación de arquitectura y tecnologías


HTML5 & CSS3 & bootstrap

Con HTML5, CSS3 y Bootstrap se desea proporcionar una apariencia agradable y amigable con el usuario. Además se quiere aprovechar los beneficios de la aplicación web, hacerla responsive y que puede ser accedida desde dispositivos móviles.

MySql

Se decidió utilizar la base de datos MySql debido a la experiencia adquirida en cursos anteriores por parte de los integrantes en esta base de datos. También porque se integra muy bien las otras herramientas y el ambiente de desarrollo escogido.

PHP

Este lenguaje de programación fue escogido por la facilidad para el desarrollo de páginas web que ofrece. Además que los integrantes del equipo han desarrollado anteriormente en esta herramienta. Además por el soporte, comunidad y facilidad de integrar se con otras herramientas

JQuery & Ajax

Esta herramienta se utiliza para la comunicación de las páginas HTML y la lógica de la aplicación por la facilidad al obtener los valores de elementos del código HTML como por ejemplo los forms. Así mismo es un complemento más estético para evitar recargar la página en cada solicitud.

Cloud 9

Este ambiente de desarrollo permite pair programing y el acceso en tiempo real al código, lo cual es de gran beneficio para la programación en grupo. Entre otros beneficios que proporciona están: servidor, base de datos MySql, PHP, Javascript y herramientas de control de versiones git.


Arquitectura

La arquitectura de la aplicación se divide en 4 capas:

Capa de vista

Esta capa es la encargada del aspecto visual de la aplicación, además de captura de datos por parte del usuario.

Capa comunicación de Javascript

Esta capa es la encargada de establecer la comunicación entre el código HTML y el controlador. Además que nos permite realizar consultas y hacer actualizaciones en el HTML sin necesidad de recargar la página.

Capa Controlador

Esta capa es la encargada de recibir la información del Ajax. Es el intermediario entre la vista y el modelo de datos. Tiene control sobre las peticiones y respuestas que se realizan. Los beneficios de esta capa es agregar una abstracción más y tener el código ordenado.

Capa Modelo

En esta capa estarán las clases de los objetos necesarios para manejar el modelo del negocio. Además es la encarga de escuchar las peticiones del controlador y responder estas peticiones. Tambien es responsable de la comunicación con la base de datos, para esto utiliza archivos para establecer la conección y posteriormente responder las peticiones del controlador.
